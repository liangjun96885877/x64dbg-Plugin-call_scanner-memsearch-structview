# MyPlugin - CALL Scanner

x64dbg 插件，用于扫描模块中的 CALL 指令并统计 API 调用次数。

## 版本

- **插件名称**: MyPlugin
- **版本**: 1.0

## 入口

- 菜单: **Plugins** → **MyPlugin** → **CALL Scanner(&W)**
- 菜单: **Plugins** → **MyPlugin** → **Memory Search(&M)**

---

## 功能概述

### 1. 模块选择

- 顶部下拉框枚举当前调试进程中已加载的所有模块
- 显示格式: `模块名 [基址, Size: 大小]`
- 切换模块后自动扫描该模块的 CALL 目标

### 2. CALL 扫描

- 遍历选中模块的可执行段（`.text`、`CODE` 等）
- 使用 `DbgDisasmFastAt` 逐条反汇编
- 识别所有 `call` 指令，提取目标地址
- 按目标地址去重，并按地址排序显示

### 3. 列表列说明

| 列名 | 说明 |
|------|------|
| # | 序号 |
| Count | 调用次数（Hook 后断点命中时累加） |
| Hook | 是否已 Hook（Yes 表示已设断点） |
| Call From (VA) | call 指令所在地址 |
| Target (VA) | 调用目标地址 |
| Target Module | 目标所在模块名 |
| API / Label | 解析出的函数名/标签 |
| Instruction | 原始反汇编指令 |

### 4. 列头排序

- 点击列头可按该列排序
- **Count** 列：首次点击按调用次数降序（高到低），再次点击切换升序
- 其他列（Call From、Target、Module、API、Instruction）：首次点击升序，再次点击切换降序

### 5. 双击跳转

- 双击 **Call From (VA)** 列 → 跳转到 call 指令地址
- 双击 **Target (VA)** 及右侧列 → 跳转到目标地址
- 自动聚焦 CPU 窗口

### 6. 多选

- 支持 Ctrl/Shift 多选行
- 右键菜单对选中行批量生效

### 7. 右键菜单

| 菜单项 | 功能 |
|--------|------|
| **Hook** | 在选中行的目标地址设置断点，开始统计调用次数 |
| **Unhook** | 删除选中行目标地址的断点，停止统计 |
| **Clear Count** | 将选中行的调用次数清零 |

### 8. 调用次数统计

- Hook 后，每次执行到目标地址时断点命中
- 插件累加计数并自动执行 `run` 继续调试
- 仅对通过 ListView 设置的 Hook 进行统计，不影响其他断点

### 8.1 自动取消 Hook

- 顶部编辑框 **Unhook when count >** 可设置阈值（默认 100）
- 当某 API 的调用次数达到或超过该值时，自动删除断点并停止统计

### 9. 断点控制

- ListView 完全控制其管理的断点
- Hook 时若该地址已有断点，会先删除再重新设置
- Unhook 时直接删除断点
- 插件卸载时自动删除所有由插件设置的断点

## Memory Search（类似 Cheat Engine）

用于在调试进程中搜索内存，支持多种扫描类型、修改前后对比、撤销扫描等，适合定位游戏/程序中的动态数据地址。

### 界面说明（布局类似 Cheat Engine）

- **左侧**：扫描结果列表 + 监控列表
  - 扫描结果：Address、Value、Previous、Module
  - **加入监控**：将选中结果加入监控列表
  - **手动添加地址**：在值框输入地址(十六进制)后添加
  - 监控列表：冻结、地址、值、类型；右键可切换冻结、修改值、删除
- **右侧**：扫描控制面板（垂直排列）
  - 数据类型（含字符串 ANSI/UTF-8/Unicode）、值、Hex、首次类型、~值2
  - 仅可写内存、**模块过滤**（全部/指定模块）
  - 二次筛选、首次扫描、二次扫描、撤销扫描、重置

### 使用流程

1. **首次扫描**：选择数据类型，输入值（Hex 勾选时输入十六进制），选择首次类型，点击 **首次扫描**
   - **字符串**：选择 字符串(ANSI)、字符串(UTF-8) 或 字符串(Unicode)，在值框输入要搜索的文本（默认不包含结束符，可匹配子串）
   - **模块过滤**：选择「全部」或指定模块名，仅扫描该模块内存
   - **未知初始值**：不输入值，直接扫描全部地址，适合先改数据再二次筛选
   - **范围**：在 ~值2 输入范围上限
2. **修改目标数据**：在目标程序中触发数据变化
3. **二次扫描**：选择筛选条件，点击 **二次扫描** 缩小范围
4. 可点击 **撤销** 回到上一次扫描结果
5. 双击或右键 **跳转到地址** 在 Dump 中查看
6. 右键 **写入值**：在值框输入新值后写入选中地址
7. **监控**：点击「加入监控」或「手动添加地址」将地址加入监控列表；右键「切换冻结」可锁定数值（每 100ms 写回）

### 字符串搜索搜不到时

- 取消勾选「仅可写内存」（字符串常在只读段如 .rdata）
- 模块过滤选「全部」
- 确认编码：程序用 GBK 选 ANSI，用 UTF-8 选 UTF-8，用宽字符选 Unicode
- 尝试更长或更独特的搜索词

### 列表说明

| 列名 | 说明 |
|------|------|
| # | 序号 |
| 地址 | 虚拟地址 |
| 值 | 当前值 |
| 之前 | 上次扫描时的值（用于对比） |
| 模块 | 所属模块 |

## 使用流程

1. 启动 x64dbg，加载或附加目标程序
2. 打开 **Plugins** → **MyPlugin** → **CALL Scanner**
3. 在下拉框中选择要分析的模块（如主程序 exe）
4. 等待扫描完成，查看 CALL 目标列表
5. 右键选择需要统计的 API，点击 **Hook**
6. 运行程序，Count 列会实时更新
7. 需要时使用 **Clear Count** 清零，或 **Unhook** 停止统计

---

## 技术说明

- 基于 x64dbg Plugin SDK
- 使用 `Script::Module::SectionListFromAddr` 获取模块段
- 使用 `DbgDisasmFastAt` 快速反汇编
- 使用 `DbgGetLabelAt` / `DbgGetModuleAt` 解析符号
- 使用 `Script::Debug::SetBreakpoint` / `DeleteBreakpoint` 管理断点
- 使用 `CB_BREAKPOINT` 回调统计命中次数
